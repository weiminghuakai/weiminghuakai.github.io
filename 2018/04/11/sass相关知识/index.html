sass相关知识

前端 sass 2018/04/11

sass

1. Sass本质就是在CSS的语法的基础上增加了自定义的变量、循环、分支、函数、mixin、继承、运算等功能，让CSS编程变得异常强大。

sass编译环境

Sass代码的编译转换工具可以用sass命令行工具、gulp、开发工具插件、webpack等。

  • 例如VScode的使用：安装Live Sass Compiler插件，然后重启完VScode之后按Ctrl+shift+p搜索sass，然后点击Live Sass:Watch Scss,这样scss文件一发生变化就自动转换为css文件
  • Sass的变量必须是$开头，后面紧跟变量名，而变量值和变量名之间就需要使用冒号(:)分隔开（就像CSS属性设置一样）。

sass的局部变量和全局变量

变量支持块级作用域，嵌套规则内定义的变量只能在嵌套规则内使用（局部变量），不在嵌套规则内定义的变量则可在任何地方使用（全局变量）。将局部变量转换为全局变量可以添加 !global 声明。

sass的特殊变量

如果变量作为CSS属性或在某些特殊情况下等则必须要以 #{$variables}形式使用

sass的嵌套(Nesting)

嵌套规则 (Nested Rules)

Sass 允许将一套 CSS 样式嵌套进另一套样式中，内层的样式将它外层的选择器作为父选择器。

sass的父选择器 & (Referencing Parent Selectors: &)

在嵌套 CSS 规则时，有时也需要直接使用嵌套外层的父选择器，可以用 & 代表嵌套规则外层的父选择器。

& 必须作为选择器的第一个字符，其后可以跟随后缀生成复合的选择器。

@at-root 用来跳出当前选择器嵌套

sass的数值运算

Sass支持数字的加减乘除、取整等运算 (+, -, *, /, %)，如果必要会在不同单位间转换值。

Sass语句

if语句

当 @if 的表达式返回值不是 false 或者 null 时，条件成立，输出 {} 内的代码。

@if 声明后面可以跟多个 @else if 声明，或者一个 @else 声明。如果 @if 声明失败，Sass 将逐条执行 @else if 声明，如果全部失败，最后执行 @else 声明

for循环语句

  • @for 指令可以在限制的范围内重复输出格式。类似for循环。

两种格式：

  • @for $var from through :条件范围包含 与 的值
  • @for $var from to : 围只包含 的值不包含 的值

each循环

@each 指令的格式是 $var in , $var 可以是任何变量名，比如 $length 或者 $name，而 是一连串的值，也就是值列表。

复杂对应each

混合指令 (Mixin Directives)

不带参数的简单Mixin

混合指令（Mixin）用于定义可重复使用的样式，可以直接把一整段Sass代码替换到某个地方去。

带参数的Mixin

Mixin还可也添加参数，跟函数一样使用

函数指令 (Function Directives)

虽然Mixin功能强大，但是某些场景下还是函数应用更方便，Sass也提供了自定义函数的用法。

继承(扩展)

sass中，选择器继承可以让选择器继承另一个选择器的所有样式，并联合声明。使用选择器的继承，要使用关键词 @extend，后面紧跟需要继承的选择器。

导入文件

Sass的导入( @import)Sass文件规则和CSS的有所不同，编译时会将 @import的scss文件合并进来只生成一个CSS文件。
但是如果你在Sass文件中导入css文件如 @import ‘reset.css’，那效果跟普通CSS导入样式文件一样，导入的css文件不会合并到编译后的文件中，而是以 @import方式存在。

所有的Sass导入文件都可以忽略后缀名 .scss。一般来说基础的文件命名方法以_开头，如 _mixin.scss。这种文件在导入的时候可以不写下划线，可写成 @import “mixin”。
