git命令以及利用girhub搭建网站

观后感 git 搭建网站 2018/04/12

命令清单（持续更新）

  • cd:切换目录

    cd ./xxx/xxxx

  • nvm ls 查看nvm的版本
  • nvm use XXX 使用XXX版本

    nvm use 5.5.0 使用5.5.0版本

  • node -v 查看当前使用的node版本
  • npm -v 查看当前使用的npm版本
  • .exit 退出node环境
  • clear清空控制台
  • ls查看当前目录的所有文件
  • cd ../ 切换到上一级目录
  • git init 初始化一个空的仓库
  • git status 查看所有文件状态
  • git add –all
  • git commit -m ‘initial commit’ 初始化提交

github上传文件步骤：

  1. 打开git-bash.exe程序

  2. 切换到需要上传文件的目录：

    切换目录 cd /E/Practice/git-demo

  3. github创建项目名称

  4. 例如：将文件上传到这个项目下面

    git remote add origin https://github.com/weiminghuakai/git-demo.git

  5. git remote -v查看分支（可有可无）

  6. git log查看日志（可有可无）

  7. git push -u origin master 上传文件到主分支并填写用户名和密码

  8. git pull origin master 将上传到github的文件下载查看

git切换增加的分支：（利用这个做网页）

  1. 查看有哪些分支：git branch
  2. 增加一个叫做v2的分支：git branch v2
  3. 再次查看有哪些分支：git branch
  4. 切换到v2分支：git checkout v2
  5. 将v2分支添加到GitHub：git push -u origin v2 然后输入用户名和密码
  6. 改变本地文件
  7. 查看当前状态：git status
  8. 将被修改过或已删除文件和所有未被跟踪的文件信息添加到索引库：git add –all
  9. 然后git commit -m ‘v2版本的第一次提交’（某种程度上相当于git add和git commit -m的组合技，前提是被改动文件已经是tracked）
  10. 再次与远端同步一下：git push -u origin v2 然后输入用户名和密码

将网站托管到GitHub并加上域名

  1. git init 初始化一个空的仓库
  2. git status查看未被跟踪的文件
  3. git add –all
  4. git status
  5. git commit -m ‘initial commit’初始化提交

  6. 上面的操作都是提交在本地，下面是

    在服务端在创建一个仓库,然后再将地址粘贴进来 git remote add origin https://github.com/weiminghuakai/XXX.git

  7. git push -u origin master 提交用户名和密码

  8. 创建新的分支 git branch gh-pages
  9. 切换到分支 git checkout gh-pages
  10. 切换到远端的分支 git push -u origin gh-pages 输入用户名和密码
  11. 例如：github的访问地址是：weiminghuakai.github.io/XXX
  12. 绑定域名
  • 创建CNAME的文件，将需要绑定的域名放在里面 比如ccvtt.com
  • 域名解析步骤：主机记录：XXX 记录类型：CNAME 记录值：github地址（weiminghuakai.github.io）然后保存就行了
  1. 将CNAME文件提交
  • github status
  • git add –all
  • git commit -m ‘添加域名绑定’
  1. push到远端：git push -u origin gh-pages 输入用户名和密码
