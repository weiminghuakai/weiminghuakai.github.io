vue Project（1）

1.代码管理

首先先找一个项目的托管仓库，将项目代码同步到远端，此处以“码云”为例

第一步：创建项目

第二步：本地代码和线上代码仓库通过git做关联，这样本地代码和线上就会同步保存

  • 在本地安装git

    • git version 检验安装是否成功（若安装成功会出现版本号）

  • 本地git和线上git关联

    • 点击设置输入命令（）

    • SSH公钥

    • 在git-bash中输入命令（ssh-keygen -t rsa -C “xxxxx@xxxxx.com“），生成公钥和私钥

    • 在git-bash中输入命令（cat ~/.ssh/id_rsa.pub），生成公钥

    • 将生成的公钥复制到SSH图中的公钥输入框中

    • 本地会有一个私钥，这样本地和线上就打通了，代码可以通过命令进行同步

  • 将线上项目同步到本地（通过ssh的网址）

    • 运行命令（git clone git@gitee.com:ccvtt/Travel.git），这样就将项目同步到本地

  • 用命令工具（cli）构建vue项目

  • 项目构建过程

  • 构建完之后会提示输入命令

  • 将本地代码同步到码云

    • git status
    • git add .
    • git commit -m ‘project initial’
    • git push

git add . 指的是将本地文件提交到本地的git缓冲区

2.项目目录讲解

  • node_modules存放依赖包

  • src存放项目的源代码

    • assets 存放图片资源
    • components存放的是项目中的小组件
    • router下面的index.js文件存放项目中所有的路由配置
    • main.js是整个项目的入口文件
    • app.vue是项目的最原始的根组件，也就是整个应用的根组件

  • config目录放的是项目的配置文件

    • index.js基础的配置信息
    • dev.env.js开发环境的信息
    • prod.env.js线上环境的配置信息

  • build目录存放的是项目打包时webpack的配置信息 （由vue-cli自动构建好的webpack的配置集合）

  • static存放静态资源

  • index.html首页的模板文件

3.单文件组件与Vue中的路由

路由是根据网址的不同，返回不同的内容给用户

@符号指的是 src这个目录

路由的配置都放在router文件夹下的index.js中，app.vue是整个应用的根组件
